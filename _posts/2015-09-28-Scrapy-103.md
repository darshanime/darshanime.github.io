Up till now, we have scraped [`dmoz`](http://www.dmoz.org/){:target="_blank"}, which was Scrapy's version of **`Hello World !`** Next was [CarDekho](http://www.cardekho.com){:target="_blank"} where we scraped data off multiple pages and stored the output in **`csv`** files. This time, we will scrape data from a website using JavaScript heavily for eg. [Housing](http://www.housing.com){:target="_blank"}.

______
<br>
The truth is that Scrapy doesn't handle JS very well. Actually, it doesn't handle JS at all. That is a problem because many modern websites use JS to dynamically generate content based on the user's interaction with the page. 

If you go to Housing and look for places to rent in Mumbai, you will notice that the URL of the page doesn't change as you move from one page to the next. How then is the client getting the data to display on the next page? 

The answer is that Housing (and most modern websites) use AJAX under the hood to request the data from the server. The task of making the request, receiving the data and displaying it correctly is done by JavaScript. Since Scrapy cannot execute JS, we cannot make those AJAX calls and thus, not get data off Housing as easily as we were able to with Cardekho.

______
<br>
There are workarounds of course, for eg. [PhantomJS](http://phantomjs.org/){:target="_blank"} which can be thought of as a full fledged **GUI-less** browser. It understands JS and can be controlled via the command line. We can use it to solve the problem at hand. Selenium is an option too, but it is very slow and unnecessarily harsh on the host.

However, it would be better if we could keep things simpler. Getting data off Housing can be done without all that complexity. Let's see how. 

______

Fire up Housing and search for places to buy in Delhi. At the time of writing, this is what the [page](https://housing.com/in/buy/search?f=eyJwb2x5Ijp7ImlkIjoiZjk3Zjk0N2ZmYWU2NDA4YWMyOTUiLCJidWZmZXIiOmZhbHNlfX0%3D){:target="_blank"} looks like :<br><br>

Look closely and you can find that that dump of text is JSON (as can also be seen on the headers for this request). It carries information about the entries on Page 2. 

Under the hood, this JSON is parsed by JS and rendered dynamically, so the user never has to navigate to a new page.

JSON is human readable as you would have noticed. The above entry is all the information (and more !) that is rendered by JS on the page. Verify that this is indeed the first entry on page 2.  

Note that entries starting with underscore( **`_`** ) are private variables and not meant for the end user. **`_score`** is one such variable. Apparently, Housing scores its listings and this one is awarded **`17.29`**. What factors the score depends on and what it signifies makes for a good guess game !

______
<br>
Once we get this JSON, we can easily parse it using Python's builtin **`json`** utility. But there is one problem, how to get the JSON ?

This one is easy. Look carefully at the **`Request URL`** which gave us the JSON. 

`https://buy.housing.com/api/v1/buy/index/filter?poly=f97f947ffae6408ac295&results_per_page=30&`**p=2**`&resale_total_count=30045&np_total_count=2329`

Now, navigate to page three and monitor the Network console for a request to **`buy.housing.com`** like before. This time the **`Request URL`** is :

`https://buy.housing.com/api/v1/buy/index/filter?poly=f97f947ffae6408ac295&results_per_page=30&`**p=3**`&resale_total_count=30045&np_total_count=2329`

*Noticed something* ? 

The URL very explicitly mentions the page number on which the data is to be displayed. 
So, the **`Request URL`** for page 53 is :

`https://buy.housing.com/api/v1/buy/index/filter?poly=f97f947ffae6408ac295&results_per_page=30&`**p=53**`&resale_total_count=30045&np_total_count=2329`


Now that we know the URL which gives us the JSON, the rest is easy as pie. We have 30 listings on each page. Total 32, 374 listings. So, we have about 1080 pages of JSON to fetch.

______
<br>
Starting a new Scrapy project :

**`$ scrapy startproject housing`**

Writing the **`items.py`** :

{% highlight python %}
from scrapy import Item, Field

class HousingItemBuy(Item):
    ad_id = Field()
    ad_title = Field()
    ad_price = Field()
    ad_area = Field()
    ad_url = Field()
    ad_date_added = Field()
    ad_coordinates = Field()
    ad_bedrooms = Field()
    ad_toilets = Field()
    ad_gas_pipeline = Field()
    ad_lift = Field()
    ad_parking = Field()                                   
    ad_gym = Field()
    ad_swimming_pool = Field()
    ad_city = Field()
    ad_locality = Field()
    ad_contact_persons_name = Field()
    ad_contact_persons_number = Field()
    ad_contact_persons_id = Field()
    count = Field()

{% endhighlight %}

<br>
Write the spider :

{% highlight python %}

from housing.items import HousingItemBuy
from scrapy import Spider
from scrapy.http.request import Request

#To parse the JSON received
import json

class HousingSpider(Spider):
    name = "housing"
    allowed_domains = ["housing.com"]
    custom_settings = {'USER_AGENT' : 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_10_1) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/40.0.2214.111 Safari/537.36'}
    

    def start_requests(self):
        #We have 1080 pages to fetch
        for count in range(1,1081):
            
            print "Getting page : %s" %count
            
            yield Request("https://buy.housing.com/api/v1/buy/index/filter?poly=f97f947ffae6408ac295&results_per_page=30&p=" + str(count) + "&resale_total_count=30045&np_total_count=2329", self.parse_buy)

           
    def parse_buy(self, response):
        
        #Since the response is purely JSON
        text =  response.body

        #Parsing it using the builtin json utility
        parsed_json = json.loads(text)
        
        #For each entry, we will store all the information we defined earlier in items.py
        #The parsed json can be read as a dict. Examining the JSON, we can easily navigate 
        #to where we have the data we need.

        for iter in range(30):
            item = HousingItemBuy()
            item['ad_price'] = parsed_json["hits"][iter]["formatted_price"]
            item['ad_url'] = parsed_json["hits"][iter]["inventory_canonical_url"]
            item['ad_title'] = parsed_json["hits"][iter]["title"]
            item['ad_coordinates'] = parsed_json["hits"][iter]["location_coordinates"]
            item['ad_date_added'] = parsed_json["hits"][iter]["date_added"]
            item['ad_area'] = parsed_json["hits"][iter]["inventory_configs"][0]["area"]
            item['ad_bedrooms'] = parsed_json["hits"][iter]["inventory_configs"][0]["number_of_bedrooms"]
            item['ad_toilets'] = parsed_json["hits"][iter]["inventory_configs"][0]["number_of_toilets"]
            item['ad_contact_persons_number'] = parsed_json["hits"][iter]["contact_persons_info"][0]["contact_no"]
            item['ad_contact_persons_id'] = parsed_json["hits"][iter]["contact_persons_info"][0]["profile_id"]
            item['ad_contact_persons_name'] = parsed_json["hits"][iter]["contact_persons_info"][0]["name"]
            
            #Some entries do not have the ad_city/ad_locality variable. 
            try:
                item['ad_city'] = parsed_json["hits"][iter]["display_city"][0]
            except :
                item['ad_city'] = "None given"
                
            try:
                item['ad_locality'] = parsed_json["hits"][iter]["display_city"][1]
            except :
                item['ad_locality'] = "None given"
                
            item['ad_gas_pipeline'] = parsed_json["hits"][iter]["inventory_amenities"]["has_gas_pipeline"]
            item['ad_lift'] = parsed_json["hits"][iter]["inventory_amenities"]["has_lift"]
            item['ad_parking'] = parsed_json["hits"][iter]["inventory_amenities"]["has_parking"]
            item['ad_gym'] = parsed_json["hits"][iter]["inventory_amenities"]["has_gym"]
            item['ad_swimming_pool'] = parsed_json["hits"][iter]["inventory_amenities"]["has_swimming_pool"]
            item['ad_id'] = parsed_json["hits"][iter]["id"]
            yield item

{% endhighlight %}
